# Flight Alert Documentation Method

This folder is for durable project guidance, not run logs, one-off task prompts, or temporary evidence. Specific examples belong here only when they explain a repeatable method.

## Durable References

- `flightalert-performance-metrics.xlsx` - performance methodology and optimization evidence that is intentionally kept durable.
- `code-organization.md` - package ownership and file-size guidance.
- `performance-debugging.md` - external probing/profiling guidance for optimization work.
- `snake-case-migration.md` - naming convention methodology and verification commands.

## Documentation Rules

- Keep durable optimization methodology in `flightalert-performance-metrics.xlsx`.
- Keep generated screenshots, videos, traces, and frame outputs out of `docs/`.
- Update durable docs when methodology, source policy, safety rules, build expectations, or user-facing behavior changes.
